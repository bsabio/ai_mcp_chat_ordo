# Master Outline: The Studio Ordo Corpus

## How to Read This Document

Each entry follows the format:
- **Chapter title** — one-sentence thesis
- **Voice** — which narrator dominates (Architect / Machine / Dialogue)
- **Emotional beat** — what the reader should FEEL at the end of this chapter
- **Assets** — visual or audio assets required
- **Armory** — reading list entries that appear in this chapter

### The 11pm Test
Every chapter is tested against: *Would the 38-year-old woman in Newark, who just got laid off, keep reading this at 11pm?* If no, the chapter fails.

### Tonal Benchmark
The stroke scene in `08_multimedia_pipeline.md` is the quality standard. If a chapter is flat by comparison, it needs revision.

### Voice Switch Triggers
The voice switches between Architect and Machine are not random. Every switch serves one of five dramatic purposes:
1. **Evidence trigger:** Architect makes a claim → Machine provides the data
2. **System trigger:** Architect describes a need → Machine shows the tool that meets it
3. **Emotional trigger:** Machine explains something cold → Architect adds the human weight
4. **Fourth-wall trigger:** Narrative becomes self-referential → Machine steps out of the story
5. **Contradiction trigger:** One voice says something the other challenges — productive tension
